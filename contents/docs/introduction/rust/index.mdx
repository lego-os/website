要开发一个操作系统内核，首先需要选择一门合适的编程语言。操作系统内核的开发对语言有着特殊的要求：它需要能够直接操作硬件，
提供对底层资源的精细控制，同时生成的代码需要高效，能够满足性能和资源的严格限制。此外，语言本身的成熟度、可移植性以及社
区支持也是重要的考量因素。

## 历史悠久的C语言

Linux 内核主要使用 C 语言编写，其他主流操作系统的代码中也大多包含大量的 C 语言部分。作为一门拥有 50 年发展历史的经典语言，
C 语言凭借其稳定性、高性能以及能够直接与硬件交互的特点，在底层软件开发领域依然占据主导地位。然而，由于缺乏内置的内存安全机制，
C 语言开发的项目往往容易出现内存安全漏洞，即便是经验丰富的开发人员，也难以完全避免这些问题。

## Rust语言

Rust 是一门专为安全性、性能和并发性设计的系统编程语言，它通过所有权和生命周期机制，从编译阶段就能有效避免绝大多数的内存安全问题。
此外，作为一门新兴的系统级语言，Rust 拥有许多现代高级语言的特性，例如更高的抽象能力、面向对象、泛型、函数式编程、更强大的宏系
统以及便捷的包管理工具 Cargo。这些特性不仅提升了代码的可读性和维护性，还极大地简化了开发过程。

目前，已有许多软件项目采用 Rust 进行重构，业界实践表明，使用 Rust 编写的程序在内存安全性方面显著优于 C 语言。同时，Rust 
的性能表现与 C 语言几乎持平，这进一步验证了其在系统级开发中的潜力。因此，我们也计划尝试使用 Rust 语言完成代码开发任务，以期获得
更安全、更高效的开发体验。
